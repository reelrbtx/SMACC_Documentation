103 ## 0.2.0 - 2018-01-29
104 
105 Requires Panda system version >= 1.1.0
106 
107 ### Motion and control interfaces
108 
109  * Improved external torque tracking behavior.
110  * Fixed discontinuities in commanding orientation changes via the cartesian
111  pose interface.
112  * Added `joint_p2p_insufficient_torque_for_planning`, `tau_j_range_violation`, and
113  `instability_detected` errors to `franka::RobotState`
114  * Added `tau_J_d`, `m_ee`, `F_x_Cee`, `I_ee`, `m_total`, `F_x_Ctotal`, `I_total`,
115  `theta` and `dtheta` to `franka::RobotState`
116  * Added `setFilter` command to `franka::Robot`
117  * Added support for commanding elbow positions for Cartesian motions.
118  * Added stiffness frame `K` to `franka::Model`
119  * **BREAKING** Replaced `p_min` and `p_max` of `franka::VirtualWallCuboid` with `object_world_size`
120 
121 ### Error handling
122 
123  * **WARNING** Not all robot errors can be recovered using the guiding button
124  on the robot anymore. To manually recover from such errors, please use the
125  the Franka DESK web interface.
126  * Added logs to `ControlException` and improved exception messages.
127  * Fail earlier (by throwing exception) if any of the commanded values are
128  `NaN` or `infinity`.
129 
130 ### Examples
131 
132  * Added saturation to joint velocity, joint position, and joint impedance
133  examples.
134  * Changed examples to read initial states inside control loops (after
135  controller switching).
136  * Examples first move to an initial joint position.
137  * Added new cartesian impedance and force control examples.
138  * Lowered grasping force in `grasp_object` example
139 
140 ### Library
141 
142  * **BREAKING** New build-time dependency on Eigen3.
143  * Changed thread priority to the maximum allowed value.
144  * Prepare for the removal of the socket-init in the default constructor in
145  POCO releases >= 1.8.0.
146  * Removed unnecessary public dependencies for libfranka.
147  * CI: Run linter on examples
148  * Docu: Use SVG instead of MathML for math rendering in API documentation to support Chrome.
